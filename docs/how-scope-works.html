Local-first project tracking

How Scope works

A kanban for epics, stories, and bugs — built so coding agents and humans can plan and track work together without ever leaving the repo.

CLI · every command speaks --json
Web UI · drag-and-drop board + swimlanes
Hub · live SSE over LAN
Event-sourced · git-syncable log
The pieces

Four faces, one .scope/ directory

Everything lives in your repo as SQLite (WAL mode) — offline, git-syncable, no external service.

⌨️

The CLI

workspace · ticket · epic · link · status · branch · pr · board — and every one supports --json so agents read and write state directly.

🗂️

The Web UI

GitHub-Projects-style columns, a ticket drawer with inline edit, and swimlanes that group by epic, assignee, priority, or type. Live dot = SSE health.

📡

The Hub — scope serve

One daemon serves UI + REST + SSE on https://localhost:4321. Self-healing & federated: hubs auto-discover, share workspaces, and promote a survivor if the owner dies.

📓

The event log

Every change is an append-only JSON file under .scope/events/. scope.db is just a cache rebuilt from it — so the workspace travels with git clone and syncs offline.

The data model

Workspaces, epics, and a status pipeline

WORKSPACE My App key: MA · one SQLite db
EPIC Auth refactor MA-1
STORY OAuth login MA-2
BUG CSRF on /signup MA-3

Status flows left to right

backlog→ todo→ in_progress→ in_review→ done

Priority

low medium high urgent

Relations (inverse auto-created)

blocks · blocked_by · relates_to
duplicates · duplicate_of

IDs like MA-3 are immutable — the prefix is baked in at create time.

How a change travels

Event log → cache → every viewer

write → log (truth) replay → cache hub fan-out (live) dumb file sync (offline)
Why it's different

Collaborate by git pull — deploy nothing

01

The log is the truth

Every change is one immutable JSON file named by a time-sortable ULID. scope.db is just a cache rebuilt from it.

02

Parallel writers never collide

Globally-unique filenames mean two people (or agents) never touch the same file. Merging is just the union of files.

03

Deterministic conflict resolution

Field edits: last-writer-wins by ULID. New tickets/comments: grow-only union. Numbers de-collide at replay.

04

Live is an optimization

Run scope serve for sub-second SSE updates — but the log stays the source of truth, online or off.

# commit the log — never the cache
$ git add .scope/events
$ git commit -m "scope: plan auth work"
$ git pull # teammates' event files

# next command rebuilds scope.db
$ scope board
↳ replayed 142 events · 0 conflicts

# want it live? one daemon, LAN-wide
$ scope serve
→ https://scope.local:4321
